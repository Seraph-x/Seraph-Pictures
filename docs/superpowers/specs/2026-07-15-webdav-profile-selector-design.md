# WebDAV 上传目标选择器设计

## 背景

`webdav.html` 当前把上传目标固定为 `storageMode: "webdav"`，文件上传和 URL 上传都不发送具体的 `storageId`。当管理员配置多个 WebDAV 存储时，页面无法指定实际上传目标，状态卡也只展示旧的全局 WebDAV 状态。

仓库已经提供管理员 Storage API、按配置测试连接的接口，以及通用上传页使用的默认配置选择和按类型记忆机制。本功能复用这些能力，不新增后端存储模型或兼容回退。

## 目标

- 在 WebDAV 上传页选择一个已启用的 WebDAV 配置。
- 文件上传、批量文件上传和 URL 上传均绑定到所选配置。
- 状态卡检测并展示当前所选配置的连接状态。
- 记住用户选择；选择失效时明确回退到默认配置。
- 在没有可用配置或配置加载失败时阻止上传并暴露原因。

## 非目标

- 不在 WebDAV 页面创建、编辑、启用或删除存储配置。
- 不迁移或重写整个 WebDAV 页面。
- 不改变 WebDAV 适配器、Storage API 或上传响应格式。
- 不把检测失败静默转换为成功，也不回退到旧全局配置。

## 方案选择

### 采用：独立 WebDAV 配置选择控制器

新增小型、可单元测试的控制器模块，负责配置加载、启用过滤、默认与记忆选择、状态检测和上传目标快照。`webdav.html` 只增加选择器标记、样式入口及少量协调调用。

选择该方案的原因：

- 复用现有 Storage API 和选择语义。
- 避免继续扩大现有单文件脚本的职责。
- 文件上传和 URL 上传可以共享同一目标快照规则。
- 核心逻辑可独立测试，不依赖完整浏览器页面。

### 未采用：直接内联到 `webdav.html`

改动较少，但会扩大既有单文件复杂度，选择与上传绑定逻辑难以独立验证。

### 未采用：迁移到通用上传组件

长期一致性较好，但涉及整页重构和更大的视觉、行为回归范围，不符合本次最小需求。

## 界面设计

### 位置

选择器位于顶部工具栏中间区域：在“当前目录”状态之后、“复制全部链接”和“清空记录”按钮之前。该位置对应用户在现有页面截图中标注的红框。

桌面端选择器自适应占用中间剩余宽度。窄屏时选择器独占一行，避免压缩当前目录和右侧操作按钮。

### 内容

- 标签：`WebDAV 目标`。
- 选项仅包含 `type === "webdav"` 且 `enabled === true` 的配置。
- 显示配置名称；默认配置附加“默认”标识。
- 配置列表加载中、无可用配置和加载失败均显示明确状态。
- 中英文文案跟随页面现有语言切换机制。

### 状态卡

“WebDAV 状态”卡片保留原位置。状态内容和“刷新状态”按钮始终绑定当前所选配置，不再使用旧的全局 WebDAV 状态。

## 组件与职责

### WebDAV 配置选择控制器

控制器负责：

- 调用管理员 Storage API 读取配置列表。
- 过滤已启用 WebDAV 配置。
- 解析记忆选择、默认配置和失效回退。
- 持久化当前选择。
- 对当前配置调用按 ID 的连接测试。
- 生成不可变上传目标快照。
- 向页面暴露明确的加载、空列表、错误、选择和检测状态。

控制器不负责 DOM 结构、文件队列、网络上传或结果列表渲染。

### 页面协调层

页面负责：

- 将控制器状态渲染到选择器和状态卡。
- 在选择变化时触发持久化和连接检测。
- 在上传开始时向控制器请求目标快照。
- 把快照字段加入现有文件与 URL 上传请求。
- 根据配置加载状态启用或禁用上传按钮。

### 现有 Storage API

复用现有接口：

- `GET /api/storage/list`：列出管理员存储配置。
- `POST /api/storage/:id/test`：测试当前选择的 WebDAV 配置。

401 处理沿用现有管理员 Storage API 行为，跳转登录并保留显式认证错误。

## 选择规则

初始化顺序：

1. 读取全部配置并过滤已启用 WebDAV。
2. 若记忆 ID 仍属于可选配置，恢复该配置。
3. 否则选择已启用的默认 WebDAV。
4. 若记忆 ID 存在但已失效，显示一次回退提示。
5. 若没有已启用的默认配置，则选择第一个已启用 WebDAV。
6. 若不存在已启用 WebDAV，进入明确的空列表状态。

选择记忆使用版本化、本地持久化结构，并沿用现有按存储类型记忆的语义。页面不得记忆凭据或配置内容。

## 上传数据流

### 文件上传

用户点击“开始上传”时生成一次不可变目标快照：

```text
storageMode = "webdav"
storageId = 当前配置 ID
storageName = 当前配置名称
folderPath = 当前规范化目录
```

同一批文件共享该快照。即使用户在上传期间切换选择器，进行中的批次也不改变目标；后续新批次使用新的选择。

每个文件的现有 `FormData` 增加准确的 `storageId`，同时保留 `storageMode` 和 `folderPath`。

### URL 上传

点击“上传 URL”时独立生成同结构快照。JSON 请求包含：

```json
{
  "url": "<remote-url>",
  "storageMode": "webdav",
  "storageId": "<selected-profile-id>",
  "folderPath": "<normalized-folder>"
}
```

## 状态检测

- 初始化选择后立即测试当前配置。
- 用户切换配置后立即测试新配置。
- “刷新状态”仅重新测试当前配置，不重载列表或改变选择。
- 较早请求晚于新请求返回时，不得覆盖新选择的状态；检测结果必须与发起时的配置 ID 对齐。
- 连接检测失败显示具体错误，但不禁用上传。真实上传仍可尝试，并由现有上传错误路径呈现结果。

## 禁用与错误状态

### 禁用上传

以下情况禁用文件上传和 URL 上传：

- 配置列表正在加载。
- 配置列表加载失败。
- 没有已启用的 WebDAV 配置。
- 当前没有有效选择。

连接检测失败不禁用上传。

### 错误展示

- 配置加载失败：选择器区域显示明确错误，不回退旧全局配置。
- 空列表：显示“没有可用的 WebDAV 配置”。
- 记忆失效：选择默认配置并显示一次回退提示。
- 状态测试失败：状态卡显示当前配置名称及具体错误。
- 上传失败：沿用现有结果和状态错误展示。
- 认证失败：沿用管理员登录跳转，不吞掉 401。

## 并发与一致性

- 配置列表和控制器对外状态使用不可变快照。
- 上传请求不得在循环中重新读取当前选择。
- 状态检测结果只有在配置 ID 仍与当前选择一致时才可提交。
- 上传进行中允许切换选择器，但切换只影响新上传。
- 配置重新加载后若当前选择失效，重新执行选择规则并明确提示。

## 测试计划

### 单元测试

- 只暴露已启用 WebDAV 配置。
- 恢复有效记忆选择。
- 无记忆时选择默认配置。
- 记忆失效时回退并产生提示。
- 无默认时选择第一个已启用配置。
- 无配置和加载失败状态禁用上传。
- 连接检测失败不禁用上传。
- 旧状态检测响应不能覆盖新选择。
- 目标快照不可变并包含准确 ID、名称和目录。

### 页面契约测试

- 选择器位于顶部工具栏的指定区域。
- 窄屏选择器换行且不挤压操作按钮。
- 文件上传 `FormData` 包含准确 `storageId`。
- URL 上传 JSON 包含准确 `storageId`。
- 一个文件批次只生成一次目标快照。
- 切换配置会记忆选择并测试新配置。
- 刷新状态只测试当前配置。
- 中英文文案完整。

### 回归验证

- WebDAV 定向测试。
- Storage API 和存储配置选择测试。
- 前端生产构建。
- 完整自动化测试集；任何既有失败与新增失败分开报告。

## 验收标准

- 管理员可以在 WebDAV 上传页选择任一已启用 WebDAV 配置。
- 文件上传、批量上传和 URL 上传到准确选择的配置。
- 所选配置在刷新页面后恢复；失效时明确回退。
- 状态卡始终对应当前选择，且不会被过期请求覆盖。
- 没有可用配置或配置加载失败时不能发起上传。
- 连接检测失败时仍允许真实上传尝试。
- 桌面和窄屏布局均保持可用。
